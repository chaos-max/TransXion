# txsim - 交易数据生成与仿真流程

该项目面向大规模资金交易数据的生成与仿真，采用"规则驱动 + 语言模型规划"的混合思路：
规则保证可控性与一致性，规划增强场景多样性与宏观合理性，适合论文描述与实验复现。

## 研究背景与目标（可直接写入论文）

现实世界的资金交易数据具有强隐私属性与高度敏感性，公开可用的数据集规模有限，
难以满足复杂实验的训练、评估与消融需求。与此同时，交易系统普遍呈现出多层次、
多尺度的统计规律：在宏观层面，交易量随时间与节假日波动；在中观层面，不同区域、
不同群体存在明显异质性；在微观层面，个体行为呈现出偏好记忆、重复交易与偶发异常。
因此，需要一种既能保持统计合理性、又能提供可控调参与实验复现能力的合成数据生成框架。

本项目的目标是提供一种面向金融交易场景的大规模合成数据生成与仿真框架，
以"宏观规划 + 微观采样"的方式复现多层结构，兼顾：
1) 规模可控；2) 行为可解释；3) 结构可调；4) 结果可复现。
该框架强调将"规划"与"落地"分离：规划阶段只负责给出分布层面的指导，
而单笔交易的细节生成完全由规则采样完成，从而保证数据一致性与可控性。

## 系统组成（概念层面）

- **数据画像层**：账户与商户画像、地理区域划分、基础偏好与活跃度分布。
- **调度与时间推进层**：以固定时间窗口推进，控制总体规模与节律。
- **宏观场景规划层**：按天给出交易量、区域强度、结构比例的趋势建议。
- **细粒度生成层**：在局部约束下生成具体交易记录。
- **落地与统计层**：记录、分片保存、统计更新、状态快照与续跑支持。

## 详细流程（论文可用版本）

1. **初始化数据与结构**
   - 载入或生成账户与商户画像，包含地域、偏好、平均交易规模与活跃度。
   - 构建区域子图与热门节点集合，为后续"本地/跨区"选择与热点偏好提供基础。
   - 初始化统计计数器与采样权重，形成可复现的随机基线。

2. **时间推进与日级场景规划**
   - 仿真按固定时间窗口推进（例如按小时或更细）。
   - 每日开始时调用规划模块生成宏观场景，包括：
     - 当日总体交易量的放大或收缩。
     - 区域强度的差异化调整。
     - 支付方式、交易规模分布的整体偏移。

3. **交易预算分配（窗口级）**
   - 将当窗口需要生成的交易数量分配到各区域。
   - 分配权重来自区域人口规模、活跃度与规划建议的联合影响。
   - 若存在场景修正因子，则对各区域预算做二次调整。

4. **交易生成（个体级）**
   - **发起方选择**：按活跃度或历史偏好加权采样，确保"活跃者更频繁"。
   - **对手选择**：在本地与跨区之间以概率切换；倾向历史偏好与热门节点。
   - **字段生成**：为每笔交易采样时间戳、金额、币种、支付方式、手续费等字段。
   - **规划与规则的分工**：规划只影响"分布与比例"，细节由规则采样完成。

5. **校验、写入与统计更新**
   - 对每笔记录执行字段完整性与格式校验，剔除不合规条目。
   - 记录写入缓冲区，达到阈值后按日期分片落盘。
   - 统计指标实时更新，包括交易类型比例、跨行比例、支付方式分布与商户热度。

6. **反馈与行为演化**
   - 历史统计反哺到后续采样，使"常见行为更易重复、热点逐渐凸显"。
   - 形成稳定与突发并存的动态模式，符合真实交易系统的演化特征。

7. **状态保存与可续跑**
   - 定期保存状态快照与统计日志，支持中断后继续生成。
   - 账户信息、映射关系与运行日志可用于实验复现与审计。

8. **结果评估与可视化**
   - 输出结构化交易表与统计汇总结果。
   - 可生成图网络与时间序列分析，用于论文展示与质量评估。

## 方法描述（正文段落版）

本项目采用分层生成策略构建交易数据。首先，在初始化阶段构建账户与商户画像，
并根据地理与群体属性形成多个区域子结构。每个账户包含基础的行为画像指标，
如平均交易频次、典型交易额度以及偏好特征等，这些指标作为后续采样的先验分布。
同时建立初始的热点集合与偏好记忆，用以刻画"交易更容易在熟悉对手之间发生"的现象。

其次，在时间推进层面，以固定时间窗口推进模拟，形成稳定的时序节律。
系统在每日开始时产生宏观场景，用以表达当日整体交易量的上升或下降趋势，
以及不同区域强度的相对变化。该宏观场景不直接生成交易，只提供分布层面的引导，
例如提高某区域的交易密度、调整支付方式比例或改变金额尺度分布等。

第三，系统在每个时间窗口内，根据区域规模与活跃度分配当期交易预算。
这一过程确保交易数量与区域结构之间保持合理比例，同时允许场景对分配进行偏移，
从而在长期尺度上形成可控的区域差异与动态波动。

第四，交易生成采用"微观规则采样"的方式完成：从账户池中采样发起方，
并根据偏好记忆、跨区概率与热点倾向选择对手。随后对时间戳、交易金额、币种、
支付方式与手续费等字段进行采样。该过程既保留随机性，又通过先验约束保持结构合理。

最后，所有交易记录将通过完整性校验，剔除不合规条目后写入缓冲区，
并按日期分片落盘。同时更新统计与偏好状态，形成持续的自反馈机制，
使得高频交易与热点节点随着时间不断强化，模拟真实系统的演化特征。

## 正文扩展：分层生成框架的动机与设计原则

从方法学角度看，交易数据生成面临两类常见矛盾：一是"可控性与多样性"的矛盾，
二是"宏观一致性与微观合理性"的矛盾。若完全依赖规则，虽然容易施加约束，
但难以表达丰富的日级场景变化，生成结果往往趋于机械；若完全依赖生成式模型，
则容易出现字段不一致、约束被破坏或不可复现的问题。为兼顾上述需求，
本项目采用分层式设计：在宏观层面由规划模块给出趋势性的分布指导，
在微观层面由规则采样严格落地每笔交易的字段与约束，从而将"创造性"与"可控性"解耦。

该框架遵循三个设计原则。第一，**约束优先**：任何与交易一致性相关的约束
（例如字段完整性、币种与金额的数值关系、手续费规则、跨行逻辑等）均由规则保证，
而非由规划模块直接生成。第二，**统计对齐**：宏观规划输出不直接决定单笔交易，
而是通过调整概率分布影响整体统计，从而更易与真实数据的边际分布对齐。
第三，**可复现与可审计**：生成过程提供状态快照与日志记录，
能够支持续跑、复现实验与事后审计，满足研究对可重复性的要求。

## 正文扩展：时间模型与规模控制

时间推进采用固定窗口机制：将全年或更长时间跨度划分为连续的时间窗口，
每个窗口对应一个生成批次。在该设置下，系统可以在全局层面规定总交易规模，
并通过窗口权重将交易量分配到不同时间段，使得数据呈现出日内周期性与跨日波动。
窗口权重可被理解为一种"时间强度函数"，用于刻画交易活跃度在一天中的变化，
例如白天更活跃、深夜更稀疏。宏观场景规划进一步在日尺度上施加乘子，
使得某些日期整体交易量上升或下降，从而模拟节假日、促销活动或系统波动带来的影响。

这种"总量约束 + 分时分配"的策略带来两个优势：其一，总规模严格可控，
便于在不同实验之间保持数据量一致；其二，时间结构可被独立调节，
便于进行消融研究，例如固定总量但改变日内节律，观察模型对时间模式的敏感性。

## 正文扩展：群体异质性与区域结构

交易系统的一个核心特征是群体异质性：不同账户的活跃度、交易额度、偏好对象
以及消费/经营属性存在显著差异。为此，系统在初始化阶段为每个账户分配行为画像，
并以区域为单位组织账户集合。区域结构不仅用于刻画"同域更频繁、跨域较少"的经验规律，
也为并行化生成提供天然的分割方式：每个区域在每个窗口内可独立生成一批交易，
最终在落地阶段统一合并与校验。

从生成结果来看，区域划分有助于形成中观层面的异质性：不同区域的交易密度、
热点商户、跨行比例与支付方式分布可以在宏观规划的引导下出现差异。
这种差异性对于研究跨域迁移、域自适应以及图结构鲁棒性等问题具有重要意义。

## 正文扩展：交易网络的形成机制（偏好记忆与探索）

真实交易网络往往呈现"重复关系 + 偶发探索"的共存结构：个体常与少数固定对手
重复交易，但也会不定期出现新的对手与新的行为模式。系统通过两类机制复现该现象。
第一类机制是偏好记忆：对每个发起方维护一个有限大小的偏好列表，
记录其历史上更常交互的对手，并随时间施加衰减，使近期行为更具影响。
第二类机制是探索：在选择对手时引入一定概率的"探索分支"，
即不从偏好列表中选择，而是从更广泛的候选集合中随机抽样，
并可由宏观规划对跨域概率进行微调。

两类机制共同作用，使得生成网络在结构上既有稳定的重复边（利于形成社群与簇），
也会持续产生新边（利于保持长时间跨度下的连通性与新颖性）。
在统计层面，这种机制通常会带来重尾度分布与热点节点的自然浮现，
同时避免网络过早收敛到少数固定模式。

## 正文扩展：字段生成与一致性约束

每笔交易记录由若干关键字段构成，覆盖时间、主体、金额、币种、渠道等维度。
字段生成遵循"先结构、后数值"的顺序：先确定发起方与对手、是否跨域、是否跨行、
采用何种支付方式，再生成金额与手续费等数值字段。金额通常围绕发起方画像中的典型额度
进行随机波动，并通过手续费规则将"支付金额"与"到账金额"关联起来，
保证二者之间存在合理差异。对于多币种场景，系统可按设定比例生成跨币种交易，
并引入汇率与价差机制，使兑换后的到账金额具有更符合现实的波动。

需要强调的是，规划模块只改变分布层面的倾向，例如提高某类支付方式的占比、
或让大额交易的比例上升。所有数值一致性与约束关系由规则采样与校验确保，
避免出现字段冲突或违反财务常识的记录。

## 正文扩展：落地、分片保存与状态快照

在生成侧输出交易列表之后，系统执行统一的落地过程：首先进行字段完整性校验，
剔除缺失字段或格式不正确的记录；随后写入内存缓冲区，并在达到阈值时触发落盘。
落盘采用按日期分片的方式，将交易记录拆分到多个文件中，
降低单文件体积，便于后续以日期为单位并行分析或增量处理。

为了支持长时运行与中断恢复，系统会周期性保存状态快照与统计日志。
状态快照记录仿真推进到的时间位置、已生成的交易量以及偏好记忆等动态信息，
使得在中断后可以从最近一次快照继续生成而无需从头开始。
统计日志则用于记录运行过程中的关键指标，便于论文实验的复现与过程诊断。

## 统计一致性与可控性说明

为保证合成数据的统计一致性与可解释性，系统在多个层面施加约束：
一方面，交易规模、跨区比例、跨行比例、交易类型比例等宏观指标可被显式控制；
另一方面，微观采样仍保留随机波动，使单笔交易在遵循约束的同时具有自然变异。
该设计避免了"完全规则化"导致的僵化模式，也降低了"完全随机化"导致的不稳定性。

此外，系统提供了可重复的实验控制能力。通过固定随机种子与保存状态快照，
可在中断后继续生成，或在同一配置下复现实验结果。
这种可复现性对于实验对比与消融研究尤为重要。

## 关键机制（更具体但不涉及实现细节）

- **交易类型结构**：个人-商户、个人-个人、商户-商户、商户-个人等类型比例可控。
- **跨区与跨行行为**：通过基准概率与规划偏移实现可控的"同城优先、异地少量"结构。
- **金额与手续费**：金额围绕画像均值随机波动；手续费按币种与跨行规则生成。
- **支付方式分布**：支持多种支付通道，规划可对其比例进行偏移。

## 正文扩展：并行化生成与效率考虑

当模拟规模达到百万级交易时，效率与资源占用成为关键因素。该框架在设计上采取两类优化。
第一类优化是"采样加速"：对发起方采样使用预计算的加权采样结构，
避免每次采样都线性扫描全体账户；对偏好列表保持有限大小与排序，
确保对手选择的代价与列表大小相关，而非与全体候选规模相关。
第二类优化是"任务并行"：在窗口级生成时，区域之间相互独立，
可在不破坏一致性的前提下并行生成各区域交易；落地阶段再统一校验与写入，
从而在保证确定性逻辑的同时提高吞吐。

在复杂度上，生成阶段的主要成本与交易条数近似线性相关，
附加成本来自偏好更新与统计更新，且都被限制在较小的常数范围内。
因此，该框架能够在保持丰富结构的同时，满足大规模合成数据的生成需求。

## 生成数据的可解释结构

在宏观层面，生成数据呈现出清晰的时序周期性与规模控制特征，
例如日内高峰与低谷、跨日的波动幅度以及区域间的密度差异。
在中观层面，交易网络形成热点与长尾并存的结构：少数节点拥有较高的收款频次，
多数节点保持低频交易，体现了现实系统中常见的幂律或重尾现象。
在微观层面，个体交易行为体现出"偏好记忆"与"偶发探索"的平衡，
既能反映稳定关系，也具备新的交易对手与支付方式的出现。

## 适用于论文的方法学表述（可直接使用）

本研究采用分层式合成生成框架，以宏观规划与微观采样相结合的方式生成交易数据。
宏观规划在日尺度上给出交易量、区域密度与交易结构的趋势性引导；
微观采样在窗口尺度内完成发起方、对手与交易字段的生成，
并通过偏好记忆与热点机制模拟真实交易网络的演化。
生成过程在每个时间窗口内执行校验与统计更新，从而形成闭环的自反馈体系。
该框架兼具可控性、可解释性与可复现性，能够支持多种规模与结构的实验需求。

## 适用于论文的"方法"章节写作模板（长文本，可直接粘贴）

本节介绍一种用于合成资金交易数据的分层生成与仿真框架。整体思路是将交易数据生成拆分为
宏观层与微观层：宏观层负责给出日级趋势与结构性约束，微观层负责在约束下生成满足一致性的
具体交易记录。该拆分使得系统既能产生具有现实感的结构变化，又能保持字段一致、过程可控与
结果可复现。

在宏观层，框架以日为单位描述场景变化。对于每一日，宏观模块输出当日交易总量的相对变化、
区域密度的增减趋势以及支付方式与交易规模分布的偏移方向。该输出不直接指定每一笔交易，
而是作为概率分布的调整项，影响后续窗口级预算分配与字段采样。宏观层的优势在于能够以低频
控制高维统计结构：通过少量的"趋势参数"即可诱导生成结果呈现合理的时间波动与区域异质性，
同时避免直接生成导致的不可控误差累积。

在微观层，框架以固定时间窗口推进仿真。每个窗口内，根据宏观层给出的总量趋势与区域强度，
首先将该窗口应生成的交易数量分配到各区域。该分配机制综合考虑区域内的主体规模与活跃度，
使得交易数量与人口规模保持一致性，并允许宏观层对其施加偏移以模拟突发活动或区域差异。
随后，在每个区域内独立生成交易记录：先从账户集合中采样发起方，再在候选对手中选择接收方，
并生成时间戳、金额、币种与支付方式等字段。

为了刻画真实交易网络的演化规律，微观层引入偏好记忆与探索机制。偏好记忆记录发起方近期更常
交互的对手，并随时间衰减，使近期行为更具影响；探索机制以一定概率跳出偏好列表，从更广泛的
候选集合中选择对手，以维持网络的新边生成与长期多样性。通过上述机制，生成网络同时包含稳定
的重复关系与持续出现的新关系，从而在结构上呈现热点与长尾并存的特征。

在字段生成方面，框架遵循"约束优先"的原则。金额围绕主体画像中的典型额度进行随机波动，
并通过手续费规则将支付金额与到账金额关联起来，保证数值关系合理。对于多币种交易，系统按设定
比例引入跨币种交易，并基于汇率与价差生成兑换后的到账金额。支付方式从预设集合中抽样，
其概率分布可由宏观层调整。由于所有一致性约束均由规则采样与校验保障，生成结果能够保持字段完备、
数值关系一致与规则可解释。

生成结果在落地阶段进行统一校验并按日期分片保存。分片策略降低单文件体积，便于后续并行分析与
增量处理。为支持长时运行与实验复现，系统周期性保存状态快照与统计日志：快照记录时间推进位置、
生成进度与偏好状态，支持中断恢复；统计日志记录交易类型比例、跨域/跨行比例、支付方式分布与热点
演化等指标，便于对生成过程进行诊断与在论文中呈现可解释结果。

总体而言，该分层生成框架在可控性、多样性与可复现性之间取得平衡，能够在不同规模与结构假设下
生成满足实验需求的合成交易数据，为风险检测、图表示学习、时序建模与鲁棒性评估等研究提供统一的数据
生成基座。

## 评估与对齐建议（论文写作可用）

为验证合成数据的合理性，可从时间、结构与主体异质性三个维度评估。
在时间维度，可比较日内强度曲线、跨日交易量波动与窗口内事件到达间隔的分布；
在结构维度，可评估交易网络的度分布、聚类系数、连通性与社群结构稳定性；
在主体维度，可对比不同群体的交易频次分布、金额分布与支付方式偏好差异。
若存在真实数据参考，可在保持隐私的前提下对齐边际分布与关键统计量，
例如对齐日级交易量、跨行比例、典型金额分位数与热点集中度等。

建议在论文中同时报告"边际一致性"和"结构一致性"两类指标：
边际一致性关注单变量或低阶统计（如金额分布与支付方式分布），
结构一致性关注网络与时序结构（如重尾度分布与周期性）。
此外，可通过消融实验展示各模块的贡献，例如移除宏观规划或移除偏好记忆，
比较时间波动、热点集中度与网络新边比例的变化，从而验证分层设计的必要性。

- **交易结构**：跨区比例、跨行比例、跨币种比例、交易类型比例。

## 局限性与合规提示（论文中建议说明）

合成数据并不等价于真实数据。尽管框架能够在多个维度上复现统计规律与结构特征，
但仍可能缺失真实世界中的制度性约束、业务规则细节与外部经济环境影响。
因此，使用者应在论文中明确说明合成数据的生成假设，并避免将结果直接泛化为真实业务结论。

同时需要强调，合成数据生成的目标是"研究可用与结构合理"，而非对任何真实个体的复现或反推。
在对齐真实数据统计量时，应仅使用聚合统计信息，避免引入任何可识别信息或敏感字段。
该框架用于研究与教学情境时，应遵循相关法律法规与伦理规范。

## 快速开始

### 环境要求

- Python 3.8+
- CUDA 11.8+ (用于vLLM推理)
- 至少1个GPU (建议8GB+显存)

### 安装步骤

1. **克隆仓库**
```bash
git clone <repository_url>
cd txsim
```

2. **创建conda环境 (推荐)**
```bash
conda create -n txsim python=3.10
conda activate txsim
```

3. **安装依赖**
```bash
pip install -r requirements.txt
```

4. **安装AgentScope**
```bash
pip install agentscope
```

### LLM服务配置

系统需要配置LLM服务。有两种方式：

#### 方式A: 使用API服务 (推荐，默认)

编辑 `LLMGraph/llms/default_model_configs.json`，填入你的API密钥:
```json
{
  "config_name": "deepseek",
  "api_key": "YOUR_DEEPSEEK_API_KEY",
  ...
}
```

`config.yaml` 中的 `llm_role_mappings` 默认已全部指向 `deepseek`，无需修改。

#### 方式B: 使用本地vLLM服务

1. **启动vLLM服务**
```bash
export MODEL_PATH="/path/to/your/Llama-3.1-8B-Instruct"
export PORT=8001
export GPU_DEVICES="0"
./manage_llm_service.sh start
```

2. **配置角色映射**
编辑 `LLMGraph/tasks/transaction/config.yaml`，将所有角色改为 `vllm_local`:
```yaml
llm_role_mappings:
  profile_generator: "vllm_local"
  merchant_generator: "vllm_local"
  transaction_manager: "vllm_local"
  scenario_generator: "vllm_local"
  default: "vllm_local"
```

3. **编辑模型配置**
编辑 `LLMGraph/llms/default_model_configs.json`，将 `YOUR_LOCAL_MODEL_NAME` 替换为实际模型路径:
```json
{
  "config_name": "vllm_local",
  "model_name": "/path/to/your/Llama-3.1-8B-Instruct",
  ...
}
```

### 运行环境检查

```bash
# 检查配置是否正确
python scripts/check_txsim_setup.py

# 或运行完整初始化测试
python scripts/check_async_init.py
```

### 启动生成

```bash
# 方式1: 使用控制脚本
./run_txn.sh start

# 方式2: 直接运行
python main_txn_async.py
```

### 配置说明

主要配置文件位于 `LLMGraph/tasks/transaction/`:

| 文件 | 说明 |
|------|------|
| `config.yaml` | 主配置: 时间范围、交易规模、LLM角色映射 |
| `scenario_calendar.yaml` | 场景日历: 节假日、促销日等 |

关键参数:
- `target_txn_count`: 目标交易数量
- `num_geos`: 地理区域数量
- `num_merchants`/`num_persons`: 每日生成的账户数量

### 输出

生成的数据保存在 `transaction_output/` 目录:
- `transactions_YYYY-MM-DD.csv`: 按日期分片的交易记录
- `state.json`: 状态快照 (用于断点续跑)
- `accounts.json`: 账户信息
